# Rolls-Royce Engineer Meeting Questions (HTGR Team, Teammate-Friendly)

Use these in the meeting exactly as written. They are phrased so anyone on the team can ask them clearly.

## 1) What accuracy levels are acceptable for final scoring?
**Question to ask:**
What error ranges are acceptable for our headline HTGR outputs, especially cycle efficiency and CO2 reduction per core MWth?

**What we need written down:**
- Acceptable error band for each metric (for example, ±X%)
- Which metrics are most important for scoring

---

## 2) What reactor-model fidelity is expected in Stage 2?
**Question to ask:**
For the HTGR second phase, is a simplified 2D unit-cell reactor model enough, or do you expect a specific benchmark/fidelity level?

**What we need written down:**
- Minimum acceptable fidelity
- Any required benchmark or preferred reference method

---

## 3) For an HTGR-only team, what minimum heat-rejection representation is acceptable?
**Question to ask:**
Since we are not an HRS team, is it acceptable for our headline model to use a fixed cooling boundary assumption (instead of a fully coupled detailed cooler), with detailed cooler results only in sensitivity/appendix?

**What we need written down:**
- Yes/no on fixed boundary as headline approach
- Any minimum checks required if we use boundary assumptions

---

## 4) Which CO2 accounting boundary should be our official headline?
**Question to ask:**
For final judging, should our main CO2 figure use operational-only accounting, or operational plus embodied impacts?

**What we need written down:**
- Official boundary mode for the headline FoM
- Whether a secondary boundary mode must also be reported

---

## 5) Which sensitivity scenarios are mandatory in the final report?
**Question to ask:**
What scenario set do you want us to report with the HTGR headline FoM (for example grid-intensity bands, process-intensity uncertainty, and cooling-auxiliary assumption range)?

**What we need written down:**
- Required scenario list
- Minimum number of scenarios
- Which scenario should be treated as the baseline
